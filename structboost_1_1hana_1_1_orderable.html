boost::hana::Orderable Struct Reference
Type classes

Description

The Orderable type class is used for data types defining a strict weak ordering.

Laws

less must define a strict weak ordering. Formally, let

x ~ y if and only if !(x < y) && !(y < x)

be the incomparability relation. Then, for all a, b, c of an orderable data type,

!(a < a) // Irreflexivity
if a < b then !(b < a) // Asymmetry
if a < b && b < c then a < c // Transitivity
if a ~ b && b ~ c then a ~ c // Transitivity of incomparability

Inherited by boost::hana::Integral::hana_enabled_operators, and boost::hana::String::hana_enabled_operators.

constexpr auto ordering
related
Initial value:
= [](auto&& f) -> decltype(auto) {
return [f(detail::std::forward<decltype(f)>(f))](auto&& x, auto&& y) -> decltype(auto) {
return less(
f(detail::std::forward<decltype(x)>(x)),
f(detail::std::forward<decltype(y)>(y))
);
};
}

Returns a function performing less after applying a transformation to both arguments.

This is not a method of the Orderable type class, but just a convenience function provided with it. Also note that

ordering(f) == less ^on^ f

Example

BOOST_HANA_CONSTEXPR_LAMBDA auto sorted = sort_by(ordering(sizeof_), tuple(
type<char[3]>,
type<char[1]>,
type<char[2]>,
type<char[15]>
));
BOOST_HANA_CONSTANT_ASSERT(sorted == tuple(
type<char[1]>,
type<char[2]>,
type<char[3]>,
type<char[15]>
));
