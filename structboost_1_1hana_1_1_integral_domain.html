boost::hana::IntegralDomain Struct Reference
Type classes

Description

IntegralDomains are commutative Rings with only zero as a zero divisor.

Typical examples of IntegralDomain include integers and polynomials over a field. The method names refer to the integral domain of integers under addition and multiplication.

Requires

Ring

Laws

For all objects a, b and k of an IntegralDomain D, the following laws must be satisfied:

mult(a, b) == mult(b, a) // commutativity
plus(mult(quot(a, b), b), mod(a, b)) == a if b is non-zero
mod(plus(a, mult(k, b)), b) == mod(a, b) if b is non-zero // canonicity
mod(zero<D>, b) == zero<D> if b is non-zero

Generalized integer remainder.

Specifically, mod(a, b) picks a canonical element of the equivalence class of a in the ideal generated by b.

Example

BOOST_HANA_CONSTANT_ASSERT(mod(int_<6>, int_<4>) == int_<2>);
BOOST_HANA_CONSTEXPR_ASSERT(mod(6, 4) == 2);

Generalized integer quotient.

Example

BOOST_HANA_CONSTANT_ASSERT(quot(int_<6>, int_<3>) == int_<2>);
BOOST_HANA_CONSTANT_ASSERT(quot(int_<6>, int_<4>) == int_<1>);
BOOST_HANA_CONSTEXPR_ASSERT(quot(6, 3) == 2);
BOOST_HANA_CONSTEXPR_ASSERT(quot(6, 4) == 1);
