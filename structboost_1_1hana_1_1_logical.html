boost::hana::Logical Struct Reference
Type classes

Description

The Logical type class is for data types acting like a boolean.

Terminology

Let x be a Logical. Then, we say that x is true-valued if and only if or_(x, y) == x for all Logicals y. Conversely, we say that x is false-valued if and only if and_(x, y) == y for all Logicals y.

Also, we say that x is a compile-time Logical if and only if it can have values of different types in both branches of the if_ method.

Conditionally return one of two values based on a condition.

Specifically, then_ is returned iff logical is true-valued, and else_ is returned otherwise. Note that some Logical instances may allow values of different types while others may require both values to have the same type.

Parameters
logicalThe condition determining which of the two values is returned.
then_The value returned when logical is true-valued.
else_The value returned when logical is false-valued.

Example

BOOST_HANA_STATIC_ASSERT(if_(true, 1, 2) == 1);
BOOST_HANA_STATIC_ASSERT(if_(false, 1, 2) == 2);
BOOST_HANA_STATIC_ASSERT(
if_(true_,
std::make_tuple('t', 'r', 'u', 'e'),
std::make_tuple('f', 'a', 'l', 's', 'e')
)
==
std::make_tuple('t', 'r', 'u', 'e')
);

Conditionally execute one of two branches based on a condition.

The selected branch will be invoked with an identity function, wich allows making types and values dependent inside a lambda and achieve a lazy-like behavior. However, type instantiation laziness can only be achieved with Integral conditions or equivalent. The result of the eval_if is the result of the invoked branch.

Parameters
logicalThe condition determining which of the two branches is selected.
then_branchA function called as then_branch([](auto x) { return x; }) iff logical is true-valued.
else_branchA function called as else_branch([](auto x) { return x; }) iff logical is false-valued.

Example (purely compile-time condition)

BOOST_HANA_CONSTEXPR_LAMBDA auto safe_make_unsigned = [](auto t) {
return eval_if(trait<std::is_integral>(t),
[=](auto id) { return id(template_<std::make_unsigned_t>)(t); },
always(t)
);
};
BOOST_HANA_STATIC_ASSERT(safe_make_unsigned(type<void>) == type<void>);
BOOST_HANA_STATIC_ASSERT(safe_make_unsigned(type<int>) == type<unsigned int>);

Example (runtime or constexpr condition)

BOOST_HANA_CONSTEXPR_LAMBDA auto safe_divide = [](auto x, auto y) {
return eval_if(y == 0,
[=](auto) { return 0; },
[=](auto id) { return id(x) / y; }
);
};
BOOST_HANA_STATIC_ASSERT(safe_divide(6, 3) == 2);
BOOST_HANA_STATIC_ASSERT(safe_divide(6, 0) == 0);

Negates a Logical.

Specifically, not_(x) returns a false-valued Logical iff x is a true-valued Logical, and a true-valued one otherwise.

Example

BOOST_HANA_STATIC_ASSERT(not_(true_) == false_);
BOOST_HANA_STATIC_ASSERT(not_(false) == true);

Return whether all the arguments are true-valued.

and_ can be called with one argument or more. When called with two arguments, and_ dispatches to the type class implementation. Otherwise,

and_(x) == x
and_(x, y, ...z) == and_(and_(x, y), z...)

Example

BOOST_HANA_STATIC_ASSERT(and_(true_, true_, true_, true_));
BOOST_HANA_STATIC_ASSERT(!and_(true_, false, true_, true_));
