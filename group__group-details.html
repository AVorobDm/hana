template<typename Algorithm >
using boost::hana::detail::by = typedef by_holder<Algorithm>

#include <boost/hana/detail/by_fwd.hpp>

Provides a .by static constexpr function object.

When creating a binary function object of type Algo whose signature is Object x Predicate -> Return, by<Algo> can be used as a base class to Algo. Doing so will provide a static constexpr member called by, which has the two following signatures:

Predicate x Object -> Return
Predicate -> (Object -> Return)

In other words, by is a curryed and flipped version of Algo. Note that the function object Algo must be default-constructible, since the algorithm will be called as Algo{}(arguments...) by by.

Note
This function object is especially useful because it takes care of avoiding ODR violations caused by the nested static constexpr member.
template<typename... Xs>
using boost::hana::detail::closure = typedef typename make_closure_impl< detail::std::make_index_sequence<sizeof...(Xs)>, Xs... >::type

#include <boost/hana/detail/closure.hpp>

Minimalistic closure implemented through multiple inheritance.

This is intended to be used as a building block for other more complex data structures. The following is guaranteed by a closure: closure<X1, ..., Xn> is a type representing a closure holding objects of types X1, ..., Xn (basically a tuple). More specifically, closure<X1, ..., Xn> is an alias to

closure_impl<
element<0, X0>,
element<1, X1>,
...,
element<n, Xn>
>

Note that this makes closure<X1, ..., Xn> a dependent type, which means that pattern matching is not allowed on it. closure_impl inherits publicly from each of these elements, which makes it possible to retrieve an object from the closure based on its index. Also note that element<n, Xn> is guaranteed to have a nested get_type alias equivalent to Xn, which is handy to retrieve that Xn without having to do pattern matching.

template<typename T , detail::std::size_t N, typename F = decltype(id)>
using boost::hana::detail::generate_integer_sequence = typedef decltype(generate_integer_sequence_impl<T, F>( detail::std::make_index_sequence<N>{} ))

#include <boost/hana/detail/generate_integer_sequence.hpp>

Generates an integer_sequence from an array-returning function.

First, F must be a constexpr-callable type and T must be a type that can be used to create an integer_sequence. The following happens when generate_integer_sequence<T, N, F> is used: a default- constructed object of type F is called with a detail::array containing the numbers in the range [0, N-1) of type T. The function F should return a detail::array, possibly with different contents. The contents of the returned array is what will end up in the final integer_sequence.

By default, when no generator function is given, generate_integer_sequence is equivalent to make_integer_sequence.
