Detailed Description

General purpose function objects.

Variable Documentation

constexpr { ... } boost::hana::_

Allows creating simple functions inline.

Common operators are overloaded to return a function object calling the corresponding operator on its arguments. Operators may also be partially applied to one argument to create Haskell-like "sections".

Overloaded operators

  • Arithmetic: binary +, binary -, /, *, %, unary +, unary -
  • Bitwise: ~, &, |, ^, <<, >>
  • Comparison: ==, !=, <, <=, >, >=
  • Logical: ||, &&, !
  • Member access: * (dereference)

Example

BOOST_HANA_CONSTEXPR_LAMBDA auto plus = _ + _;
BOOST_HANA_STATIC_ASSERT(plus(1, 2) == 1 + 2);
BOOST_HANA_CONSTEXPR_LAMBDA auto increment = _ + 1;
BOOST_HANA_STATIC_ASSERT(increment(1) == 2);
BOOST_HANA_CONSTEXPR_LAMBDA auto double_ = 2 * _;
BOOST_HANA_STATIC_ASSERT(double_(1) == 2);

template<std::size_t n>
constexpr auto boost::hana::arg
Initial value:
= [](auto ...xs) {
static_assert(n > 0,
"invalid usage of arg with n == 0");
static_assert(sizeof...(xs) >= n,
"invalid usage of arg with too few arguments");
return detail::at_index::best(size_t<n-1>, xs...);
}

Returns its nth argument.

Indexing starts at 1, so that arg<1> returns the 1st argument, arg<2> the 2nd and so on. Using arg<0> is an error. Passing less than n arguments to arg<n> is also an error.

Example

BOOST_HANA_STATIC_ASSERT(arg<2>('1', "2", 3) == "2");

constexpr auto boost::hana::compose
Initial value:
= [](auto f, auto g) {
return [=](auto x, auto ...xs) { return f(g(x), xs...); };
}

Returns the composition of two functions.

Note
compose is an associative operation; compose(f, compose(g, h)) is equivalent to compose(compose(f, g), h).

Example

BOOST_HANA_CONSTEXPR_LAMBDA auto increment = [](auto x) {
return x + 1;
};
BOOST_HANA_CONSTEXPR_LAMBDA auto double_ = [](auto x) {
return x * 2;
};
BOOST_HANA_STATIC_ASSERT(compose(increment, double_)(3) == (3 * 2) + 1);

Proof of associativity (. is compose)

f . (g . h) $ x xs... == f ((g . h) x) xs...
== f (g (h x)) xs...
(f . g) . h $ x xs... == (f . g) (h x) xs...
== f (g (h x)) xs...
template<unsigned arity>
constexpr auto boost::hana::curry

Returns a curriable function of the given arity.

Specifically, curry<n>(f) is a function which can be curried with up to n arguments. However, unlike with usual currying, a curried function may still be called with several arguments at a time; curry<n>(f)(x1, ..., xN) is equivalent to curry<n>(x1)...(xN).

Example

BOOST_HANA_CONSTEXPR_LAMBDA auto add = [](auto x, auto y, auto z) {
return x + y + z;
};
BOOST_HANA_STATIC_ASSERT(curry<3>(add)(1)(2)(3) == 1 + 2 + 3);
BOOST_HANA_STATIC_ASSERT(curry<3>(add)(1)(2, 3) == curry<3>(add)(1)(2)(3));
BOOST_HANA_STATIC_ASSERT(curry<3>(add)(1, 2, 3) == curry<3>(add)(1)(2)(3));
Note
As expected, curry<0>(f) is a function which can be curried with up to 0 arguments. Hence, it may only be called as follows: curry<0>(f)().
BOOST_HANA_CONSTEXPR_LAMBDA auto two = []() {
return 2;
};
BOOST_HANA_STATIC_ASSERT(curry<0>(two)() == two());
Todo:
Consider making curry<0>(f) equivalent to f. It might be more general or less useful if it creates subtle pitfalls.
constexpr auto boost::hana::fbind
Initial value:
= [](auto f, auto ...g) {
return [=](auto ...x) { return f(g(x...)...); };
}

Invokes f with the result of invoking each g with all the arguments.

Note
  • The arity of f must match the number of gs.
  • When used with two functions only, fbind is associative. In other words, fbind(f, fbind(g, h)) is equivalent to fbind(fbind(f, g), h).

Example

BOOST_HANA_CONSTEXPR_LAMBDA auto divide = [](auto x, auto y) {
return x / y;
};
BOOST_HANA_STATIC_ASSERT(fbind(divide, arg<2>, arg<1>)(5, 10) == 10 / 5);

Find a better name.

Proof of associativity in the binary case

bind f (bind g h) $ xs... == f ((bind g h) xs...)
== f (g (h xs...))
bind (bind f g) h $ xs... == (bind f g) (h xs...)
== f (g (h xs...))
constexpr { ... } boost::hana::fix

Computes the least fixed point of a function.

Specifically, fix(f, xs...) is equivalent to f(fix(f), xs...), where fix(f) is fix partially applied to f. fix is an implementation of the Y-combinator, also called the fixed-point combinator. It encodes the idea of recursion, and in fact any recursive function can be written in terms of it.

Example

BOOST_HANA_CONSTEXPR_LAMBDA auto factorial = fix([](auto fact, auto n) {
if (n == 0) return 1;
else return n * fact(n - 1);
});
BOOST_HANA_STATIC_ASSERT(factorial(5) == 120);

Returns a function invoking f with its two first arguments reversed.

BOOST_HANA_CONSTEXPR_LAMBDA auto minus = [](int x, int y, int z = 0) {
return x - y - z;
};
BOOST_HANA_STATIC_ASSERT(minus(3, 0) == 3 - 0);
BOOST_HANA_STATIC_ASSERT(flip(minus)(3, 0) == 0 - 3);

constexpr auto boost::hana::on
Initial value:
= [](auto f, auto g) {
return [=](auto ...x) { return f(g(x)...); };
}

Invokes f with the result of invoking g on each argument.

Note
on is associative, i.e. on(f, on(g, h)) is equivalent to on(on(f, g), h).

Examples

BOOST_HANA_CONSTEXPR_LAMBDA auto first = [](auto pair) {
return pair.first;
};
BOOST_HANA_CONSTEXPR_LAMBDA auto plus = [](auto x, auto y) {
return x + y;
};
std::pair<int, int> x{1, 2}, y{10, 20};
BOOST_HANA_STATIC_ASSERT(on(plus, first)(x, y) == 1 + 10);
BOOST_HANA_CONSTEXPR_LAMBDA auto increment = [](auto x) {
return x + 1;
};
BOOST_HANA_CONSTEXPR_LAMBDA auto multiply = [](auto x, auto y) {
return x * y;
};
BOOST_HANA_STATIC_ASSERT(on(multiply, increment)(2, 4) == 3 * 5);

Proof of associativity

on(f, on(g, h))(xs...) == f(on(g, h)(xs)...)
== f(g(h(xs))...)
on(on(f, g), h)(xs...) == on(f, g)(h(xs)...)
== f(g(h(xs))...)
